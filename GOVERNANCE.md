# Governance

Mason uses maintainer-led governance. Contributors propose and test changes;
reviewers approve areas with sustained expertise; maintainers merge, release,
handle security reports, and set compatibility and safety policy.

Changes to command execution, permissions, credentials, OAuth, checkpoints,
provider routing, session storage, public claims, licensing, or governance need
a written design issue and maintainer consensus. Published model-quality claims
must link to reproducible multi-trial evidence and disclose failed or neutral
results. Maintainer status is granted by consensus after sustained contribution.
